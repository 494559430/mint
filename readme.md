### 需求细化

基于您的描述，我将需求细化成清晰、可实现的步骤和规格，确保AI（或开发工具）能准确理解并构建静态页面。目标是创建一个简单的歌单搜索页面，使用HTML/CSS/JavaScript实现，无需后端。

#### 1. **整体功能概述**
- **页面类型**：纯静态网页（HTML + CSS + JS），可在浏览器中本地运行或部署到静态主机（如GitHub Pages）。
- **核心功能**：展示歌单，支持按标签搜索歌曲，点击歌曲名时自动复制“点歌 歌名”文本到剪贴板。
- **数据来源**：歌单数据和歌曲数据存储在单独的JSON文件中，便于维护和更新。
- **用户交互**：
  - 主页加载时，显示所有歌曲列表。
  - 搜索框：输入标签/歌名/作者等关键词，实时过滤歌曲（支持部分匹配，如输入“流行”匹配“流行音乐”）。
  - 标签展示：搜索框下方列出所有唯一标签（从歌曲数据中提取），每个标签可点击，直接填入搜索框进行过滤。
  - 歌曲列表：显示匹配的歌曲，每行包含歌名、作者、标签。点击歌名时，自动复制“点歌 [歌名]”到剪贴板，并可选显示提示（如“已复制”）。 比如我搜索出《天若有情》这首歌，点击歌曲名时，会自动复制“点歌 天若有情”到剪贴板。
- **UI/UX要求**：
  - 简洁现代：使用基础CSS样式（无框架），响应式设计（适配手机/PC）。
  - 无错误处理：假设数据格式正确，无需验证。
  - 性能：数据量小（假设<1000首歌），无需优化。

#### 2. **数据结构规格**
- **歌单数据**：单独JSON文件（`playlist.json`），包含歌单基本信息（如标题、描述）。
  -  一个歌单一个JSON文件（`playlist.json`），包含：
  - `name` (string): 歌名，必填。
  - `author` (string): 作者，必填。
  - `tags` (array of strings): 标签列表，必填，至少一个标签。
  - 示例：
    ```json
    {
      "name": "夜曲",
      "author": "周杰伦",
      "tags": ["流行", "国语", "经典"]
    }
    ,{
      "name": "天若有情",
      "author": "周杰伦",
      "tags": [ "国语", "古风"]
    }
    ```
- **数据加载**：页面启动时，JS异步加载`playlist.json`，然后加载所有歌曲JSON，合并成一个歌曲数组。

#### 3. **页面结构和组件**
- **HTML结构**（`index.html`）：
  - 头部：标题（使用歌单标题）。
  - 搜索区域：输入框（placeholder: "输入标签搜索"），下方标签列表（动态生成）。
  - 歌曲列表：`<ul>`或`<div>`，每首歌一行，显示“歌名 - 作者 (标签1, 标签2)”。
- **CSS样式**（`styles.css`）：
  - 基础布局：居中容器，搜索框和标签列表样式（按钮式）。
  - 歌曲列表：列表项hover效果，点击歌名高亮。
- **JavaScript逻辑**（`script.js`）：
  - 加载数据：使用`fetch`加载JSON。
  - 搜索过滤：监听输入框输入，过滤歌曲数组，更新列表。
  - 标签生成：从所有歌曲提取唯一标签，渲染为可点击按钮。
  - 点击复制：使用`navigator.clipboard.writeText`复制文本，兼容性考虑（fallback alert）。
  - 事件绑定：搜索框输入、标签点击、歌名点击。

#### 4. **文件结构**
- `index.html`: 主页面。
- `styles.css`: 样式文件。
- `script.js`: 逻辑文件。
- `playlist.json`: 歌单数据。
